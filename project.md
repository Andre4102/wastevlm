# Waste-VLM — project state & next steps

*Last updated: 2026-05-29.*

Custom-VLM curriculum for illegal-waste detection in aerial/drone imagery
(AerialWaste v2 m2/m4 + DroneWaste). Target architecture: **DINOv3/RADIO vision
encoder + projector + Qwen2.5-7B-Instruct LLM** (LLaVA-style). The vision encoder
is DINO/RADIO, *not* Qwen's native ViT.

---

## Active decision (2026-05-29): SKIP the LLM text-CPT stage; build a VQA dataset instead

**Rationale.** Continued-pretraining a 7B model on only ~389K tokens is low-yield
and risks denting instruction-following. The captions + corpus are far more
valuable as the *substrate* for a VQA dataset, which is simultaneously (a) the
training data for the visual stage and (b) the aerial-waste benchmark the field
currently lacks (no aerial-waste VQA benchmark exists; Waste-Bench is
ground-level only).

**Status of CPT.** Shelved, not deleted. `src/lora_cpt_llm.py`,
`src/build_cpt_data.py`, `data/waste_corpus/cpt_blocks.jsonl`, `slurm_lora_cpt.sh`
all kept. `accelerate` was bumped 0.34.2 → 1.2.1 (the trainer's only blocker), so
CPT is resurrectable in one `sbatch` if we ever want it.

## VQA dataset spec (decided)

- **Purpose:** BOTH a visual-SFT **training set** AND a held-out **eval
  benchmark** ("better than Waste-Bench") — one generation pipeline, two splits.
- **Split:** site-stratified; **reuse the existing seg 70/15/15 seed=0 split** so
  the *same sites* stay in the same partition → zero train/eval leakage. The
  eval benchmark = the held-out 15% test sites, heavily QC'd.
- **Question types (in scope for v1):**
  1. **Presence (yes/no)** — positive (gt non-empty) vs negative (gt empty);
     balanced; good negative-control.
  2. **Waste-type identification** — multi-label, **grounded in `gt_categories`
     only** (captions not used for the answer).
  3. **Spatial grounding / localization** — grounded in segmentation masks &
     derived bounding boxes (strongest on DroneWaste paper-10; AerialWaste where
     region geometry exists). This is what most waste VQA lacks.
  4. **Scenery description** — natural-language description of the visible
     setting (terrain, vegetation, water, structures, human activity).
     Generated by a **constrained agent-dispatch re-caption pass** whose prompt
     explicitly forbids mentioning waste / debris / garbage / dumping /
     pollution. The no-waste constraint makes the dangerous failure mode
     (fabricating waste types in the answer) structurally impossible — the
     worst scenery hallucination can do is mis-label landcover (e.g., cropland
     vs pasture), which doesn't bias the model toward wrong waste predictions.
- **Hallucination defense (2026-06-15 decision).** All v1 *waste-related*
  answers (Qs 1–3) are derived 100% from GT (labels + masks). The existing
  captions are **not** consulted to produce any waste-related answer the VLM is
  trained on. Rationale: GT was human-curated by multi-interpreter
  photo-interpretation; captions are agent-distilled and carry some
  hallucinations. Generating answers from GT means caption noise cannot
  propagate into the VLM's waste-detection signal. **Carve-out for Q4
  (scenery):** scenery answers can't be GT-derived because the dataset has no
  landscape labels; the safeguard is the no-waste constraint in the scenery
  re-caption prompt — any residual hallucination is in landcover phrasing
  only, never in waste content.
- **Deferred (not in v1):**
  - **Description / free-form Qs** — would require trusting caption text in
    answers; deferred until/unless we can run the faithfulness verifier at
    scale (de-scoped for the two-person team).
  - **Risk & regulatory-reasoning Qs** — would draw on the corpus + EU List of
    Waste / Commission Decision 2014/955/EU. The corpus is retained for exactly
    this later add-on.

## Resolved decisions

- **Question-phrasing strategy (2026-06-15): templates + paraphrase bank.**
  Each Q-type has a fixed template structure with a small bank of ~10–20
  agent-generated paraphrases per type, sampled uniformly at generation time.
  The paraphrase bank is built in a single one-shot agent pass (agent never
  sees images, so no per-image hallucination risk into question wording).

## Next steps (ordered — restart here)

1. **Define the unified waste-type label space.** DroneWaste paper-10,
   AerialWaste m2 (5-class) and m4 (6-class) use different taxonomies — build a
   superset/mapping so type-ID questions share one vocabulary.
2. **Build the split index.** Map every `image_id` → {train|val|test} from the
   seg site-stratified seed=0 split; assert no site spans partitions.
3. **Build the paraphrase bank.** One-shot Claude Code agent pass → ~10–20
   paraphrased question variants per Q-type (presence, type-ID, grounding,
   scenery). Output: `data/vqa/paraphrase_bank.json`. Agent sees only the
   intent of each Q-type, never an image.
4. **Generate scenery descriptions** via a constrained agent-dispatch
   re-caption pass over all 9,297 images. Prompt (locked phrasing):
   *"Describe the visible scenery: terrain, vegetation, water, structures,
   and human activity. Do NOT mention waste, debris, garbage, dumping, or
   pollution — even to deny their presence."*
   Output: `data/captions/scenery.jsonl` (keyed by `image_path`).
5. **Write the QA generator:**
   - presence → yes/no from gt emptiness, class-balanced;
   - type-ID → multi-label from `gt_categories` against the unified label space;
   - grounding → masks → bboxes (connected-components, reuse
     `seg_eval_decode.py`); region-referring questions; document the
     amorphous-waste caveat (waste is not instance-like, so expect coarse
     boxes);
   - scenery → answer pulled from `scenery.jsonl` for that `image_path`.
   - Question wording sampled from the paraphrase bank; answers for Qs 1–3
     come ONLY from GT, answers for Q4 come from the constrained scenery
     captions.
   - Output: **LLaVA-style instruction JSON** (train split) + **benchmark
     JSON with answers + image references** (eval split).
6. **Sanity-check the eval slice.** Qs 1–3 answers are correct by
   construction; QC focuses on (a) question quality (sample ~50 eval items —
   answerable from image alone, GT-derived answer matches human read), and
   (b) a quick visual spot-check of scenery descriptions on ~50 positives to
   confirm the no-waste constraint held. The 3-agent ensemble
   (`src/aggregate_votes.py`) covers edge cases.
7. **Stats + snapshot.** Counts per question type × dataset × split; add a VQA
   section to `RESULTS_SNAPSHOT.md`.
8. **Then the visual stage:** wire DINO/RADIO encoder + projector to
   Qwen2.5-7B-Instruct, train on the VQA train split, evaluate on the
   benchmark.

## Parked

- **AerialWaste multiblock-conv probe** — multiblock concatenation (K blocks → K*D
  → LR) strongly hurts DINO on AW classification (−6 to −7 pp m2 F1 vs linear)
  while being neutral for RADIO. Root cause: LR is flooded with noisy low-level
  features from early blocks on a small dataset. A conv-based head could fix this:
  extract patch tokens from K blocks as [B, K*D, G, G], apply Conv2d(K*D, 256, 3) →
  GAP → classify. This keeps the classifier input at 256-dim regardless of K, and
  the spatial conv can learn which blocks to trust. Requires a small PyTorch training
  loop (not sklearn), so it's a lightweight fine-tuned head rather than a pure probe.
  Not implemented; defer to future work.

- **Caption QA at scale** — de-scoped: no external reviewers, manual review of
  9k captions isn't realistic for two people. Captions are accepted as-is as the
  VQA substrate; the (small) **eval slice** gets QC via the existing 3-agent
  ensemble + faithfulness verifier (`src/aggregate_votes.py`,
  `inspect_captions.ipynb`).
- **17 flagged captions + 8 recaption suggestions** —
  `data/captions/flagged_captions.jsonl`, `data/captions/recaption_queue.jsonl`.
- **LLM text-CPT** — shelved (see top).
